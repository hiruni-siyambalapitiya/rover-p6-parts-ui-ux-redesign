# 🚗 Rover P6 Parts UX/UI Case Study

> A Human-Computer Interaction (HCI) case study focused on evaluating and redesigning the user experience of the Rover P6 Parts e-commerce website through usability analysis, user research, contextual inquiry, and user-centered design principles.

---

# 📌 Project Overview

This project was completed as part of the Human-Computer Interaction (HCI) module at the Sri Lanka Institute of Information Technology (SLIIT).

The objective was to evaluate the existing **Rover P6 Parts** e-commerce website, identify usability issues affecting users, and propose improvements using UX research techniques and Human-Centered Design principles.

Instead of developing a new system, the project focused on understanding user behavior, identifying usability problems, and recommending interface improvements to enhance the overall user experience.

---

# 🎯 Project Objectives

- Evaluate the usability of an existing e-commerce website.
- Identify UI/UX issues affecting different user groups.
- Conduct user research using contextual inquiry.
- Create user personas.
- Prioritize usability issues.
- Recommend user-centered design improvements.

---

# 🧩 Business Problem

The Rover P6 Parts website contained several usability problems that negatively impacted the overall user experience.

Some of the identified issues included:

- Poor navigation
- Inconsistent layouts
- Small fonts
- Unclear buttons
- Excessive white space
- Lack of responsive design
- Poor visual hierarchy
- Difficult-to-read content
- Confusing page layouts

The goal of this project was to improve usability while making the website easier and more enjoyable for users.

---

# 👥 User Groups Identified

Three primary user groups were identified during the research process:

- Unregistered Users
- Customers
- Resellers

Each group had different goals, expectations, and usability concerns.

# 📝 Contextual Inquiry

User interviews were conducted to understand user behaviors, expectations, and pain points.

Example activities included:

- Searching for products
- Registering an account
- Adding products to the cart
- Completing checkout
- Viewing ratings
- Accessing customer information
- Contacting support

---

# ⚠️ Key Usability Issues

Some of the major usability problems identified were:

## Navigation

- Difficult navigation
- Poor page organization
- Missing visual hierarchy

## Interface Design

- Small buttons
- Small text
- Inconsistent colors
- Inconsistent layouts

## Content

- Too much information
- Poor readability
- Unclear page structure

## Accessibility

- Poor mobile responsiveness
- Difficult-to-read fonts
- Poor color contrast

## User Experience

- Confusing workflows
- Large empty spaces
- Outdated interface

---

# 📊 Usability Prioritization

Usability issues were categorized based on their severity.

- 🔴 High Priority
- 🟠 Medium Priority
- 🟢 Low Priority

This helped determine which issues should be addressed first during redesign.

# 💡 Proposed Improvements

The following recommendations were made to improve the website:

- Modern UI Design
- Responsive Layout
- Improved Navigation
- Better Typography
- Consistent Color Palette
- Clear Call-to-Action Buttons
- Improved Product Pages
- Better Shopping Cart Experience
- Simplified Checkout Process
- Enhanced Visual Hierarchy
